=== v0.9 — Planned ===

[ ] Channel message attachments — post images directly in channel chat; reuse existing
    upload pipeline; store with channelId instead of eventId on MediaAsset

[ ] Thread replies — reply to a specific message; collapse into "3 replies" indicator;
    schema: add parentMessageId FK on Message

[ ] Bulk member import — admin uploads a CSV of emails; sends invite emails to each

[ ] Customisable reminder timing — per-user setting for how far in advance event_start
    push/email fires (15 min / 1 hr / 1 day / none); stored in UserNotificationPreference
    with type = "event_start_offset"

[ ] Digest emails — daily or weekly email summarising upcoming events the user is
    RSVP-yes on; scheduled via BullMQ repeatable job

[ ] In-app notification grouping — group consecutive notifications of the same type for
    the same event (e.g. "5 people RSVPed to your event")

[ ] Image captions — optional caption text field on MediaAsset; shown below photos in
    the lightbox and media grid

[ ] Album / media page per event — dedicated full-page view (/events/:id/media) with
    masonry or grid layout, download-all button, sorting by date

[ ] Offline queue for RSVPs — when user RSVPs while offline, queue action in localStorage
    and replay on reconnect (extends existing useIsOnline hook)

[ ] Event page map embed — when event.location is set, show embedded static map thumbnail
    (Google Maps Static API or OpenStreetMap tile) above the links

[ ] Accent colour per user — User.theme is a string so it can carry "dark:indigo" or
    similar; render as CSS custom property on the root element

[ ] Keyboard shortcut help modal — ? key opens overlay listing all keyboard shortcuts
    (lightbox arrow nav, Escape, etc.)

[ ] Waitlist — when maxAttendees reached, late RSVPs join a waitlist; promote automatically
    when a yes-RSVP withdraws (status: "waitlist" on RSVP)

[ ] Event comments / discussion thread — Message model already has an eventId FK and the
    relation is in the schema; wire up a simple threaded comment feed on EventPage
    (no socket needed, just paginated REST)

[ ] Event check-in QR code — generate a QR code linking to a one-click RSVP-yes endpoint;
    display on EventPage for the creator; useful for in-person check-in via phone scan

[ ] isLegendary badge — flag is in schema and returned by API but only shown as a crown
    icon; let admins toggle it on the event edit modal and surface legendary events
    highlighted on GroupPage

[ ] dev vs prod instance — separate .env.production, separate DB, Redis, and storage bucket
